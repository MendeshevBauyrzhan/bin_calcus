Данный проект представляет из себя реализацию бинарного калькулятора на LabView который может выполнять следующие 10 операций:
1) Сложение - "+"
2) Вычитание - "-"
3) Умножение - "*"
4) Деление - "/"
5) Синус - sin(x)
6) Косинус - cos(x)
7) Возведение в степень - "a^b"
8) Экспонента - "e^x"
9) Квадратный корень - sqrt(x)
10) Факториал - n!

 Структура проекта:
*VI для работы с бинарным представлением чисел:*

**split_bin_string.vi** -- из строкового представления числа в двоичной системе разделяет его на: знак (1 - отрицательный, 0 - положительный), массив целой части, массив дробоной части. Массивы хранятся в реверсивной форме, это удобно так как операции сложения, вычитания, умножения, деления выполняются справа налево.
**collect_binstr.vi** -- из знака, массивов целых и дробных чисел собирает строковое представление числа. Используется для отображения результата операций. Сами операции выполняются через массивы чисел. При этом сбор строки двоичного числа происходит с учетом того что в массивах целые и дробные части реверсированы.
**align_arrays.vi** -- выравнивает размеры масссивов целых/дробных чисел дополняя пустые места нулями. Нужно при операциях сложения, вычитания, умножения, деления так как целые и дробные части двух чисел могут не совпадать.
**max_min.vi** --  находит максимальное из двух двоичных чисел. Первый вывод соответствует максимальному числу, второй минимальному. Если числа равные выводы тоже будут одинаковы. Данный VI необходим при реализации вычитания когда необходимо установить какое число нужно вычитать от которого. 


*Непосредственно операторы калькулятора:*
**calculator.vi** -- основной файл содержащий элементы управления. Состоит из двух полей для ввода операнодов, enum для выбора операции, и индикатор для вывода результата. В случае если оператор унарный береться первый операнд, второй игнорируется;
**plus-minus.vi** -- реализует операции сложения и вычитания. Так как природа операций схожа было решено соединить их в один VI. На входе два двоичных числа. Оба числа расщепляются на массивы split_bin_string.vi. Дальше в зависимости от того являются ли их знаки одноименными или разноименными выбирается сложение или вычитание:

sign1 XOR sign2 
1) оба "+"/оба "-" => сложение
2) один "+"/другой "-" (или наоборот) => вычитание

Логика сумматора:

Сначала происходит сложение дробной части. Потом происходит сложение целой части при этом перенос разряда с дробной части передается в целую часть. При сложении происходит выравнивание массивов а также удаление избыточных нулей. Используется следующая формула сумматора:

Сумма: S = A ⊕ B ⊕ Cin
Перенос разряда: Cout = (A ∧ B) ∨ (Cin ∧ (A ⊕ B))
Здесь:
A, B -- слагаемые разряды
Cin -- перенос с прошлого разряда
Cout -- перенос в следующий разряд

Логика вычитателя реализована схожим образом но для него используется формула вычитателя. Также при вычитании нужно определить максимальное число и вычитать минимальное из него. 

Вычтенное: D = A ⊕ B ⊕ Bin
Заем: Bout = (¬A ∧ B) ∨ (¬A ∧ Bin) ∨ (B ∧ Bin)
Здесь:
A, B -- вычитаемые разряды
Bin -- заем с прошлого разряда
Bout -- заем с текущего разряда

**multiply.vi** -- реализует умножение. Происходит очищение избыточных нулей и выравнивание массивов как и в plus-minus.vi. Умножение реализуется через сложение. К примеру 5*2 = 5 + 5 = 10.
**devide.vi** -- реализует логику деления. Деление реализуется через вычитание. Сначала произодиться проверка на то что является ли делитель нулём. Если нет то начинается деление. Логику можно условно разделить на два этапа:
1) Делитель меньше или равно делимого. В таком случае отнимаем от делимого делитель и увеличиваем целую часть результата на 1.
2) Делитель меньше делимого. В таком случае умножаем делимое на 0b10 и также отнимает делитель пока не достигнем нужного количества точек после запятой (дробное деление).
